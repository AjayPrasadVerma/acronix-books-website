Acronix Books ships as a lightweight desktop app for Windows. Installation takes under a minute and needs no administrator rights.

## System requirements

| Requirement | Details |
| --- | --- |
| Operating system | Windows 10 or Windows 11 (64-bit) |
| Disk space | ~200 MB for the app; your books grow with your data |
| RAM | 4 GB minimum, 8 GB recommended |
| Internet | Only needed for download, updates and optional cloud sync |

macOS and Linux builds are in the pipeline — see the [download page](/download/) to be notified at launch.

## Download and install

1. Download the installer:
   [AcronixBooks-Setup-latest.exe](https://acronixbooks.com/updates/AcronixBooks-Setup-latest.exe) (~80 MB).
2. Double-click the downloaded file. This is a **per-user installer** — it installs into your own user profile, so Windows will not prompt for an administrator password.
3. Wait a few seconds for setup to finish. Acronix Books launches automatically.

<Callout type="info" title="SmartScreen warning?">
Early-access builds may show a Windows SmartScreen prompt because the code-signing reputation is still building. Click **More info → Run anyway**. This is expected for new applications and does not indicate a problem.
</Callout>

## First launch

On first launch you'll create a local account to secure the app:

1. Choose a username and a strong password. This password unlocks the app and is used to derive the encryption key for your books — see [Security](/docs/security/).
2. **Save the recovery code** shown to you. It's the only way back in if you forget your password. Store it somewhere safe and offline.
3. You're in. Next, [create your first company](/docs/creating-your-first-company/).

<Callout type="warn" title="Your password cannot be reset for you">
Acronix Books encrypts your data with a key derived from your password. There is no central server that can reset it. If you lose both your password and your recovery code, your local book cannot be decrypted. Keep the recovery code safe.
</Callout>

## Staying up to date

Once installed, the app **auto-updates itself** from the release feed in the background. You can also trigger a manual check any time — see [Updating the app](/docs/updating-the-app/).
