Acronix Books keeps itself current so you're always on the latest features and fixes without hunting for downloads.

## Automatic updates

Once installed, the app checks the **release feed** in the background and downloads new versions automatically. When an update is ready, you're prompted to restart to apply it — at a moment that suits you.

Because it's a **per-user install**, updates apply without administrator rights, just like the initial [installation](/docs/installation/).

## Check for updates manually

You don't have to wait for the background check:

1. Open **Settings → About / Updates**.
2. Choose **Check for updates**.
3. If a newer version exists, it downloads and offers to restart.

## Your data is untouched by updates

An update replaces the **application**, not your **books**. Your encrypted database stays exactly where it is; the new version opens it in place.

<Callout type="tip" title="Update at a natural break">
Apply updates between tasks rather than mid-invoice, and take a quick [backup](/docs/backup-and-restore/) around month-end filings. This is good hygiene for any accounting software.
</Callout>

## Seeing what changed

Every release is documented in the [Changelog](/changelog/), grouped by area — accounting, GST, inventory, security and cloud sync — so you can see exactly what's new after an update.

<Callout type="info" title="Staying on the current version matters">
Updates carry security hardening and GST-related corrections. Running the latest build is the simplest way to keep your books accurate and protected.
</Callout>
