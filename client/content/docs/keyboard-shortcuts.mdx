Acronix Books is designed so an experienced operator rarely touches the mouse. Interactions are tuned to feel instant (sub-200ms), and every screen has a **status bar that always lists the shortcuts active right now** — so you can discover the keys as you work instead of memorising them up front.

<Callout type="info" title="Discoverability is built in">
You never have to guess. The status bar at the bottom of the window shows the shortcuts that apply to the current screen, dialog or field. The keys below are the ones you'll use across the whole app.
</Callout>

## Jump straight to a module

The top-level modules are bound to the function keys, exactly like the on-screen navbar shows. Each also has an <code>Alt</code> + letter equivalent.

| Shortcut | Module |
| --- | --- |
| <code>F1</code> · <code>Alt</code> + <code>G</code> | Gateway (dashboard) |
| <code>F2</code> · <code>Alt</code> + <code>V</code> | Vouchers |
| <code>F3</code> · <code>Alt</code> + <code>A</code> | Accounts |
| <code>F4</code> · <code>Alt</code> + <code>I</code> | Inventory |
| <code>F5</code> · <code>Alt</code> + <code>R</code> | Reports |
| <code>F6</code> · <code>Alt</code> + <code>B</code> | Banking *(coming soon)* |
| <code>F7</code> · <code>Alt</code> + <code>S</code> | GST |
| <code>F8</code> · <code>Alt</code> + <code>U</code> | Audit |
| <code>F9</code> · <code>Alt</code> + <code>E</code> | Settings |

## Global keys

| Shortcut | Action |
| --- | --- |
| <code>Ctrl</code> + <code>K</code> | Open the command palette / quick search |
| <code>Esc</code> | Go back — close a dialog, pop a screen, or cancel the current field |
| <code>Ctrl</code> + <code>U</code> | Check for app updates |

## Working inside a form or voucher

| Shortcut | Action |
| --- | --- |
| <code>F10</code> | **Save** the voucher or form |
| <code>Enter</code> | Accept the current field and advance to the next |
| <code>Tab</code> / <code>Shift</code> + <code>Tab</code> | Next / previous field |
| <code>Esc</code> | Cancel and go back |

## Moving through lists and reports

| Shortcut | Action |
| --- | --- |
| <code>↑</code> / <code>↓</code> | Move the selection |
| <code>Enter</code> | Open the selected row |
| <code>Y</code> / <code>N</code> | Confirm or dismiss a yes/no prompt |

<Callout type="tip" title="Learn three, save hours">
If you only memorise three, make them <code>Ctrl</code> + <code>K</code> (find anything), <code>F10</code> (save) and <code>Esc</code> (back). Those three carry you through most of a billing day; the status bar shows you the rest as you go.
</Callout>
